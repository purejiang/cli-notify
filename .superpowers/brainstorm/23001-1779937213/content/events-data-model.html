事件类型 & 交互模型

桌面 → 手机的事件流，以及手机 → 桌面的反向操作

桌面 → 手机（事件流）

事件 来源 内容
session.start Hook: SessionStart 会话 ID、时间戳、工作目录
message.user Session 文件解析 用户消息文本
message.assistant Hook: Stop + 文件 Claude 回复文本
tool.request Hook: PreToolUse 工具名、参数、需要批准
tool.result Hook: PostToolUse 工具执行结果
session.end Hook: SessionEnd 会话结束、摘要

手机 → 桌面（反向操作）

📝 reply

发送文本消息，Desktop Agent 通过 claude --continue 注入到当前会话

✅ approve

批准工具权限请求，Desktop Agent 通过进程间通信放行

❌ deny

拒绝工具权限请求

⚡ status

查询当前会话状态、同步完整历史

关键设计问题

Claude Code 没有原生 API 接收外部输入，需要设计方案来注入手机的回复和审批指令。
