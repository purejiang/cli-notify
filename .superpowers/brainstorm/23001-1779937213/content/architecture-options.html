架构方案对比：Claude Code Remote Control

三种方案实现桌面 ↔ 手机同步交互

A

轻量 Agent 中转

桌面运行一个轻量 Node.js agent 进程，通过 Claude Code hooks 捕获事件写入本地 WebSocket server。手机在同一 WiFi 下直连，或通过 Cloudflare Tunnel 免费穿透外网访问。

优点

  • 架构最简单，一个 agent 进程搞定
  • Cloudflare Tunnel 免费且安全
  • 无云服务器成本

缺点

  • 依赖第三方 tunnel 服务
  • 桌面必须保持运行
  • 不支持离线消息
B

自建 WebSocket 中继

一个轻量 Go/Node WebSocket 服务部署在便宜的 VPS 上。桌面 agent 和手机 app 都作为 WS 客户端连入。支持消息队列、会话恢复、离线通知（FCM）。

优点

  • 消息可靠，支持离线
  • 完全自主可控
  • FCM 推送通知

缺点

  • 需要 VPS（约$5/月）
  • 运维成本
  • 数据经过云端
C

Hybrid 智能路由（推荐）

方案 A + B 的融合。桌面 agent 同时启动本地 WS server 和云端连接。手机优先尝试局域网直连（低延迟），不可达时自动切到云中继。云端作为 fallback 通道。

优点

  • 局域网零延迟体验
  • 外网无缝 fallback
  • 消息双通道保障

缺点

  • 实现复杂度最高
  • 需要 VPS + tunnel
  • 双通道状态同步需仔细设计
